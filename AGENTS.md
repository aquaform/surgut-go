# Стандарты проекта\n\n## Стек по умолчанию\n- Backend: Node.js 20 + (Fastify | Express — уточняется на discuss)\n- Хранилище: уточняется на discuss\n- Тесты: vitest (или jest), цель покрытия 80%+ на бизнес-логику\n\n## Обязательный контракт деплоя\n- В корне всегда есть рабочий Dockerfile.\n- Сервис слушает порт из переменной PORT (дефолт 3000). В Node.js обязательно использовать хост 0.0.0.0, иначе Traefik отдаст 404.\n- После первого написания кода создай GitHub-репозиторий (gh repo create <app> --public), добавь remote origin и сделай push main. Это необходимо для Dokploy!\n- Есть эндпоинт GET /health, возвращающий 200 и тело \"ok\".\n- Деплой выполняется ТОЛЬКО через слэш-команду /deploy. Вручную curl'ом в проде — нельзя.\n\n## Код-стандарты\n- Типы на всех публичных функциях.\n- Маленькие чистые модули, без god-файлов.\n- Никаких секретов в коде. Конфиг только из переменных окружения.\n\n## Чего НЕ делать\n- Не читать и не печатать .env и содержимое каталогов secrets/.\n- Не выполнять деструктивный bash (rm -rf, git push --force).\n- Не тянуть зависимости, не относящиеся к задаче.\n
